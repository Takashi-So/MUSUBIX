# チュートリアル: 備品管理システムを作る

このチュートリアルでは、MUSUBIXを使って「備品管理システム」を開発する過程をステップバイステップで解説します。要件定義→設計→タスク分解→実装→検証の各フェーズで、実際のプロンプト例と出力結果を確認しながら進めます。

## 前提条件

- MUSUBIXがインストール済み（[クイックスタート](../quickstart.md)参照）
- Claude Code または GitHub Copilot が使える環境

## チュートリアル構成

| パート | 内容 | 所要時間（目安） |
|-------|------|----------------|
| [01. セットアップ](01-setup.md) | プロジェクト初期化と環境確認 | 5分 |
| [02. 要件定義](02-requirements.md) | EARS形式での要件定義とレビュー | 15分 |
| [03. 設計](03-design.md) | C4モデル設計とレビュー | 15分 |
| [04. 実装](04-implementation.md) | タスク分解とコード生成 | 30分 |
| [05. 検証](05-verification.md) | トレーサビリティ検証と完了 | 10分 |

## 完成イメージ

このチュートリアルで作る備品管理システムの機能:

- 備品の登録（名前、カテゴリ、購入日、価格、保管場所）
- 備品の検索
- 備品の貸出・返却
- 貸出中の重複防止
- 廃棄記録・廃棄済み備品の貸出防止
